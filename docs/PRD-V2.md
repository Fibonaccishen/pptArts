# PPTArts 产品需求文档 V2.0

## 一、产品概述

### 1.1 产品定位

PPTArts 是一款面向企业内部的 PPT 组件库桌面应用。用户可以按分类浏览、搜索、预览和下载公司统一的 PPT 组件（箭头、图标、信息图框、结构模板等），也可以自行导入 PPTX/PNG/SVG 文件到库中，自动生成缩略图。

### 1.2 目标用户

公司内部员工（初期 10-20 人），日常需要撰写 PPT，希望使用统一风格组件提升效率和视觉一致性。

### 1.3 核心价值

- 集中管理公司 PPT 视觉资产，避免风格碎片化
- 按分类和标签快速找到所需组件
- 多格式导入（PPTX / PNG / SVG），自动生成缩略图，降低维护成本
- 一键下载组件源文件，粘贴即用

## 二、系统架构

```
┌─────────────────────────────────────────────────┐
│  Electron 桌面客户端 (React + TypeScript)         │
│  ├── 分类导航                                      │
│  ├── 组件浏览 & 预览（含类型标识）                   │
│  ├── 搜索                                          │
│  ├── 下载（多格式支持）                              │
│  └── 后台管理（导入、编辑、删除）                     │
└───────────────┬─────────────────────────────────┘
                │ HTTP REST API (JWT)
┌───────────────▼─────────────────────────────────┐
│  Express 服务端 (Node.js + TypeScript)            │
│  ├── 用户认证 (/api/auth)                         │
│  ├── 组件 CRUD (/api/components)                  │
│  ├── 多格式导入 & 缩略图生成                        │
│  └── 文件静态服务                                   │
└───────────────┬─────────────────────────────────┘
                │
┌───────────────▼─────────────────────────────────┐
│  数据存储                                         │
│  ├── SQLite 数据库（组件元信息、用户）               │
```

### 部署方式

- 服务端部署在云端小服务器或个人电脑
- 客户端通过 Electron 打包为 Windows .exe，分发给团队成员安装
- 客户端配置服务端地址，通过 HTTP API 通信
- 支持自动更新：electron-updater + generic HTTP provider

## 三、组件分类体系

采用**二级分类结构**，从实际使用场景出发：

| 一级分类 | 二级分类 | 使用场景 |
|---|---|---|
| **基础元素** | 箭头、人物角色、商务办公、科技数字、生活场景、分割线&装饰、过渡元素、文字框、引用框、标签、其他 | 已有页面大纲，拖入小组件丰富细节 |
| **结构模板** | 并列结构、架构图、对比图、流程图&时间轴、金字塔、循环、矩阵、信息图 | 缺乏灵感，需要一个完整页面布局填入内容 |

### 分类原则

- 一级分类按使用场景：小组件 vs 整页模板，用户进入时直觉清晰
- 二级分类按形态细分，互不交叉
- 组件可通过标签跨分类检索，弥补少数边界模糊的情况

## 四、V2 功能清单

### 4.1 用户登录

**功能描述**：简单的用户认证，保证只有团队内部人员可以访问。

**详细说明**：
- 预置账号密码，由管理员在数据库手动添加
- 登录后获取 JWT Token，后续请求携带 Token
- 无注册页面、无密码重置、无 OAuth
- 无权限分级，登录即可使用全部功能

**验收标准**：
- [x] 输入正确账号密码后进入主界面
- [x] 输入错误提示"账号或密码错误"

### 4.5 下载文件

**功能描述**：下载组件的源文件到本地。

**详细说明**：

**验收标准**：

### 4.7 文件导入系统

**功能描述**：上传 PPTX / PNG / SVG 文件导入到组件库，自动生成缩略图。

**详细说明**：

- **支持格式**：PPTX、PNG、SVG，单文件最大 50MB，一次最多 200 个文件
- **上传入口**：导航栏或管理页的"导入组件"按钮
- **上传流程**：
  1. 选择本地一个或多个文件（支持多选，支持三种格式混选）
  2. 预览上传列表（文件名 + 文件大小）
  3. 填写导入信息：
     - 目标一级分类（下拉选择）
     - 目标二级分类（根据一级联动）
     - 组件名称（默认取文件名去掉扩展名）
     - 标签（预设常用标签可勾选 + 自定义标签输入）
     - 上述信息批量应用于所有导入文件
  4. 点击"开始导入"
  5. 服务端处理（按文件类型分发缩略图管道）：
     - **PPTX**：解析 XML 获取内容包围盒 → LibreOffice 渲染整页 → 精确裁剪到内容区域 + padding → upscale
     - **PNG**：sharp 自动 trim 白边 → upscale（如需要）
     - **SVG**：sharp 内置光栅化 → trim → upscale
     - 所有缩略图统一输出为 PNG，最小尺寸 240px
  6. 显示导入进度和结果

**验收标准**：
- [x] 支持一次选择多个文件（PPTX/PNG/SVG 可混选）
- [x] 分类和标签信息正确写入数据库
- [x] PPTX 缩略图聚焦内容区域，自动裁剪空白边距
- [x] PNG/SVG 缩略图自动生成并可在网格中显示
- [x] 导入进度提示，完成/失败反馈清晰

## 六、数据模型

### 6.1 用户表 (users)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PRIMARY KEY | 自增 ID |
| username | TEXT NOT NULL UNIQUE | 用户名 |
| password_hash | TEXT NOT NULL | BCrypt 加密的密码 |
| created_at | DATETIME | 创建时间 |

### 6.2 组件表 (components)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PRIMARY KEY | 自增 ID |
| name | TEXT NOT NULL | 组件名称 |
| category | TEXT NOT NULL | 一级分类 |
| subcategory | TEXT NOT NULL | 二级分类 |
| tags | TEXT | 逗号分隔的标签 |
| pptx_path | TEXT NOT NULL | 源文件相对路径 |
| thumbnail_path | TEXT NOT NULL | 缩略图（PNG）相对路径 |
| file_type | TEXT NOT NULL DEFAULT 'pptx' | 文件类型：pptx / png / svg |
| download_count | INTEGER NOT NULL DEFAULT 0 | 下载次数 |
| created_at | DATETIME | 导入时间 |
| updated_at | DATETIME | 最后修改时间 |

### 6.3 索引

- components 表：`(category, subcategory)` 联合索引
- components 表：`name` 索引
- components 表：`download_count DESC` 索引
- 搜索使用 `LIKE` 查询 name 和 tags 字段（SQLite 对较小数据量足够）

## 七、API 设计（概要）

### 7.1 认证

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | /api/auth/login | 登录，返回 JWT Token |

### 7.2 组件

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | /api/components | 获取组件列表（支持分类筛选、排序、分页、搜索） |
| GET | /api/components/:id | 获取单个组件详情（含 file_type） |
| GET | /api/components/:id/download | 下载源文件（Content-Type 根据 file_type 动态设置） |
| POST | /api/components/import | 批量导入（multipart/form-data，支持 .pptx/.png/.svg） |
| PUT | /api/components/:id | 编辑组件信息 |
| DELETE | /api/components/:id | 删除组件 |
| POST | /api/components/batch-delete | 批量删除 |
| GET | /api/components/:id/thumbnail | 获取缩略图（公开，无需鉴权） |

### 7.3 分类

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | /api/categories | 获取全部分类树（含组件计数） |

## 九、非功能需求

### 9.1 性能

- 组件列表 API 响应时间 < 500ms（1000 个组件以内）
- 缩略图加载使用懒加载，避免一次请求过多图片
- 搜索防抖 300ms
- PNG/SVG 缩略图生成速度远快于 PPTX（无 LibreOffice 调用开销）

### 9.2 可用性

- 缩略图最小尺寸 240px，保证网格视觉整齐
- PPTX 缩略图自动裁剪到内容区域，聚焦有效内容
- 分类导航始终可见（不随滚动消失）
- 所有操作有明确的结果反馈（成功 toast / 错误提示）
- 文件类型标识在卡片、管理表格、详情弹窗中一致显示

### 9.3 数据安全

- 服务端文件存储目录不可通过 URL 直接访问
- 源文件通过鉴权 API 返回，不暴露真实路径
- 缩略图公开访问（供 `<img>` 标签加载）
- JWT Token 有效期 7 天
