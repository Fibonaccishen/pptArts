# PPTArts 产品需求文档 V1.0

## 一、产品概述

### 1.1 产品定位

PPTArts 是一款面向企业内部的 PPT 组件库桌面应用。用户可以按分类浏览、搜索、预览和下载公司统一的 PPT 组件（箭头、图标、信息图框、结构模板等），也可以自行导入新的 PPTX 组件到库中，由系统自动生成缩略图。

### 1.2 目标用户

公司内部员工（初期 10-20 人），日常需要撰写 PPT，希望使用统一风格组件提升效率和视觉一致性。

### 1.3 核心价值

- 集中管理公司 PPT 视觉资产，避免风格碎片化
- 按分类和标签快速找到所需组件
- PPTX 直接导入，自动生成缩略图，降低维护成本
- 一键下载组件 PPTX 文件，粘贴即用

```

│  ├── 用户认证 (/api/auth)                         │
```

- 服务端部署在云端小服务器或个人电脑
- 客户端通过 Electron 打包为 Windows .exe，分发给团队成员安装
- 客户端配置服务端地址，通过 HTTP API 通信

## 三、组件分类体系

采用**二级分类结构**，一级大类按组件粒度划分，二级子类按形态细分：

| 一级分类 | 二级分类 |
|---|---|
| **基础元素** | 箭头、图标库、分割线&装饰、过渡元素 |
| **内容容器** | 文字框、信息图框、数据卡片、引用框&标签 |
| **结构模板** | 并列结构、架构图、对比图、流程图&时间轴、金字塔&循环&矩阵 |

### 分类原则

- 一级分类按组件粒度：原子级 / 分子级 / 页面级，用户进入时直觉清晰
- 二级分类按形态细分，互不交叉
- 组件可通过标签跨分类检索，弥补少数边界模糊的情况

## 四、V1 功能清单

### 4.1 用户登录

**功能描述**：简单的用户认证，保证只有团队内部人员可以访问。

**详细说明**：
- 预置账号密码，由管理员在数据库手动添加
- 登录后获取 JWT Token，后续请求携带 Token
- 无注册页面、无密码重置、无 OAuth
- 无权限分级，登录即可使用全部功能

**验收标准**：
- [ ] 输入正确账号密码后进入主界面
- [ ] 输入错误提示"账号或密码错误"
- [ ] Token 过期后自动跳回登录页

---

### 4.2 三级分类导航

**功能描述**：左侧导航栏展示两级分类树，用户点击任一分类查看该分类下的组件。

**详细说明**：
- 左侧固定导航栏，宽度约 200px
- 一级分类默认展开，显示其下的二级分类
- 点击一级分类可折叠/展开
- 点击二级分类后，右侧内容区刷新为该分类下的组件
- 当前选中的分类高亮显示
- 下方有一个"全部组件"入口，不区分分类查看所有组件

**验收标准**：
- [ ] 左侧显示完整的两级分类树
- [ ] 点击一级可折叠/展开二级
- [ ] 点击二级后右侧内容区正确刷新
- [ ] 当前选中分类高亮

---

### 4.3 组件缩略图网格

**功能描述**：主内容区以网格形式展示当前分类下的组件缩略图。

**详细说明**：
- 每行固定列数（如 4-5 列），响应式适配窗口宽度
- 每个卡片包含：缩略图、组件名称
- 支持分页，每页 20-40 个组件
- 空分类时显示"暂无组件，请导入"提示

**验收标准**：
- [ ] 网格布局整齐展示缩略图和名称
- [ ] 分页功能正常
- [ ] 窗口缩放时列数自适应

---

### 4.4 组件详情预览

**功能描述**：点击组件卡片，弹出大图预览窗口。

**详细说明**：
- 点击卡片弹出模态窗口，显示高清缩略图
- 窗口内有左右箭头，可切换到同分类下上一个/下一个组件
- 显示组件名称、所属分类、标签
- 下方有"下载 PPTX"按钮
- 点击窗口外部或关闭按钮关闭预览

**验收标准**：
- [ ] 点击卡片弹出大图预览
- [ ] 左右箭头切换同分类组件
- [ ] 显示名称、分类、标签信息
- [ ] 关闭按钮正常

---

### 4.5 下载 PPTX

**功能描述**：下载组件的 PPTX 源文件到本地。

**详细说明**：
- 预览窗口和卡片上均有下载按钮
- 下载文件名为"组件名称.pptx"
- 使用系统原生保存对话框选择下载路径
- 或在应用设置中指定默认下载目录，直接下载

**验收标准**：
- [ ] 点击下载，成功保存 PPTX 文件到本地
- [ ] 文件名与组件名称一致
- [ ] 下载的 PPTX 文件可正常打开

---

### 4.6 关键词搜索

**功能描述**：通过搜索栏输入关键词，跨分类检索组件。

**详细说明**：
- 顶部搜索栏，输入关键词后回车或自动搜索（防抖 300ms）
- 搜索范围：组件名称 + 标签
- 搜索结果以网格展示，按匹配度排序
- 结果页顶部显示"搜索'xxx'的结果，共 N 个"
- 支持空格分隔多个关键词（AND 逻辑）

**验收标准**：
- [ ] 输入关键词后正确筛选组件
- [ ] 匹配名称和标签
- [ ] 搜索结果按匹配度排序
- [ ] 无结果时显示"未找到相关组件"

---

### 4.7 PPT 导入系统

**功能描述**：上传 PPTX 文件导入到组件库，自动生成缩略图。

**详细说明**：

- **上传入口**：导航栏或管理页的"导入组件"按钮
- **上传流程**：
  1. 选择本地一个或多个 PPTX 文件（支持多选）
  2. 预览上传列表（文件名 + 文件大小）
  3. 填写导入信息：
     - 目标一级分类（下拉选择）
     - 目标二级分类（根据一级联动）
     - 组件名称（默认取文件名去掉扩展名，可逐行修改）
     - 标签（逗号分隔，可选）
     - 上述信息批量应用于所有导入文件（作为默认值）
  4. 点击"开始导入"
  5. 服务端处理：保存 PPTX → 调用 LibreOffice 生成缩略图 → 写入数据库
  6. 显示导入进度和结果

- **缩略图生成**：服务端使用 LibreOffice headless 模式，将 PPTX 第一页转为 PNG

**验收标准**：
- [ ] 支持一次选择多个 PPTX 文件
- [ ] 分类和标签信息正确写入数据库
- [ ] 缩略图自动生成并可在网格中显示
- [ ] 导入进度提示，完成/失败反馈清晰

---

### 4.8 组件管理

**功能描述**：编辑和删除已导入的组件。

**详细说明**：
- 导航栏下方有"管理"入口，切换到管理视图
- 管理视图为表格形式：

| 缩略图(小) | 组件名称 | 一级分类 | 二级分类 | 标签 | 操作 |
|---|---|---|---|---|---|
- 支持编辑：名称、一级分类、二级分类、标签（行内编辑或弹窗编辑）
- 删除：点击删除按钮 → 二次确认 → 删除数据库记录 + 服务端文件（PPTX + 缩略图）
- 支持批量选择删除

**验收标准**：
- [ ] 表格正确展示所有组件
- [ ] 编辑功能正常保存
- [ ] 删除后数据库和文件均被移除
- [ ] 删除有二次确认

## 六、数据模型

### 6.1 用户表 (users)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PRIMARY KEY | 自增 ID |
| username | TEXT NOT NULL UNIQUE | 用户名 |
| password_hash | TEXT NOT NULL | BCrypt 加密的密码 |
| created_at | DATETIME | 创建时间 |

### 6.2 组件表 (components)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PRIMARY KEY | 自增 ID |
| name | TEXT NOT NULL | 组件名称 |
| category | TEXT NOT NULL | 一级分类 |
| subcategory | TEXT NOT NULL | 二级分类 |
| tags | TEXT | 逗号分隔的标签 |
| pptx_path | TEXT NOT NULL | PPTX 文件相对路径 |
| thumbnail_path | TEXT NOT NULL | 缩略图相对路径 |
| created_at | DATETIME | 导入时间 |
| updated_at | DATETIME | 最后修改时间 |

### 6.3 索引

- components 表：`category`, `subcategory` 联合索引
- 搜索使用 `LIKE` 查询 name 和 tags 字段（SQLite 对较小数据量足够）

## 七、API 设计（概要）

### 7.1 认证

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | /api/auth/login | 登录，返回 JWT Token |

### 7.2 组件

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | /api/components | 获取组件列表（支持分类筛选、分页、搜索） |
| GET | /api/components/:id | 获取单个组件详情 |
| GET | /api/components/:id/download | 下载 PPTX 文件 |
| POST | /api/components/import | 批量导入 PPTX（multipart/form-data） |
| PUT | /api/components/:id | 编辑组件信息 |
| DELETE | /api/components/:id | 删除组件 |
| DELETE | /api/components/batch | 批量删除 |
| GET | /api/components/:id/thumbnail | 获取缩略图 |

### 7.3 分类

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | /api/categories | 获取全部分类树 |

## 九、非功能需求

### 9.1 性能

- 组件列表 API 响应时间 < 500ms（1000 个组件以内）
- 缩略图加载使用懒加载，避免一次请求过多图片
- 搜索防抖 300ms

### 9.2 可用性

- 缩略图统一尺寸（如 400x300 px），保证网格视觉整齐
- 分类导航始终可见（不随滚动消失）
- 所有操作有明确的结果反馈（成功 toast / 错误提示）

### 9.3 数据安全

- 服务端文件存储目录不可通过 URL 直接访问
- PPTX 文件通过鉴权 API 流式返回，不暴露真实路径
- JWT Token 有效期 7 天
